Webhook

The payment gateway uses webhooks to deliver real-time notifications about payment events.

Webhook delivery is designed to be reliable and ordered. The gateway guarantees that notifications are delivered in the correct sequence for each individual entity (e.g. invoice or charge).

Overview​

Delivery Guarantees​

  • Webhook delivery is guaranteed
  • Event ordering is preserved per entity
  • Each event is delivered at least once. The consumer must handle webhook processing in an idempotent way, as duplicate deliveries may occur due to retries or network failures.

Retry Policy​

If a webhook delivery fails (e.g. due to a non-2xx response or timeout), the system will automatically retry delivery:

  • Maximum retry attempts: 150
  • Retry interval: 10 minutes between attempts

After all retry attempts are exhausted, the delivery is considered failed and will not be retried further.

Webhook Integration​

To receive real-time notifications from the payment gateway, you must configure a publicly accessible HTTP endpoint that can handle incoming webhook requests.

Before enabling webhooks, you must first:

  1. Create a project in the gate panel
  2. Obtain PROJECT_ID and PROJECT_SECRET
  3. Configure a webhook URL for the project

The webhook URL is the endpoint where the gateway will send all event notifications.

Once configured, the gateway will send POST requests to your endpoint whenever a relevant event occurs (e.g. invoice status change, new charge payment).

Your endpoint must:

  • Accept POST requests
  • Be accessible over HTTP/HTTPS
  • Respond with a 2xx status code within a reasonable timeout
  • Perform signature verification before processing the payload

If the endpoint does not respond successfully, the gateway will retry delivery according to the retry policy.

Webhook Payload Schema​

Each webhook request contains the following fields:

FieldTypeDescription
idstringUnique webhook notification identifier. Should be used for idempotency handling on the receiver side.
eventstringEvent type that triggered the webhook.
dataobjectEvent payload containing domain-specific data related to the event.
attemptnumberDelivery attempt number. Incremented on each retry in case of failed delivery.
timestampnumberUnix timestamp (seconds) when the webhook notification was created. Can be used for replay protection and signature validation.

Signature Verification​

To ensure webhook authenticity, each incoming request must be verified using HMAC SHA-256.

The server sends a signature in the X-Webhook-Signature header.
To validate the request, you must:

  1. Take the raw request body (exact bytes, without parsing or re-encoding).
  2. Compute an HMAC SHA-256 hash using the ZERO_PAY_PROJECT_SECRET as the key.
    The secret is provided as a hex-encoded string
  3. Compare the computed signature with the value from X-Webhook-Signature.

If the signatures do not match, the request must be rejected.

Example​

Pseudocode Example​

rawBody = getRawRequestBody()
receivedSignature = getHeader("X-Webhook-Signature")

secret = HEX_DECODE(ZERO_PAY_PROJECT_SECRET)

computedSignature = HMAC_SHA256(secret, rawBody)

if computedSignature != receivedSignature:
    rejectRequest(401)
else:
    processWebhook()

TypeScript Example (Nest.JS Guard)​

class WebhookSignatureVerifier implements CanActivate {
  protected logger = new Logger(WebhookSignatureVerifier.name)
  protected static readonly signatureHeader = "X-Webhook-Signature"
  protected static readonly secret = process.env.ZERO_PAY_PROJECT_SECRET

  public async canActivate(context: ExecutionContext): Promise<boolean> {
    const req = context.switchToHttp().getRequest<RawBodyRequest<Request>>()

    if (!req.rawBody) {
      this.logger.error(`Cannot validate webhook message because req.rawBody is undefined`)
      return false
    }

    const signatureHeader = req.header(WebhookSignatureVerifier.signatureHeader)
    if (!signatureHeader) {
      return false
    }

    const message = req.rawBody
    const computedSignature = WebhookSignatureVerifier.computeMessageSignature(
      message,
      WebhookSignatureVerifier.secret,
    )

    return crypto.timingSafeEqual(Buffer.from(signatureHeader, "hex"), Buffer.from(computedSignature, "hex"))
    // return signatureHeader === computedSignature
  }

  public static computeMessageSignature(message: Buffer, apiSecret: Buffer) {
    return crypto.createHmac(`sha256`, apiSecret).update(message).digest(`hex`)
  }
}
