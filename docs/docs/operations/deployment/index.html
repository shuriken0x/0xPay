Deployment

This section describes the minimal server preparation required before deploying the payment gateway.

Docker Installation​

Docker (and docker-compose) is required on both the host machine (used for building and deployment) and the target server (used for running services).

Both environments must also have access to the private container registry.

Installation (Host & Server)​

Run the following commands on both machines:

apt-get update

curl -sS https://get.docker.com/ | sh  

export release="v2.27.2"
sudo curl -L https://github.com/docker/compose/releases/download/$release/docker-compose-`uname -s`-`uname -m` -o /usr/bin/docker-compose 
sudo chmod +x  /usr/bin/docker-compose 

systemctl enable docker  

systemctl status docker # docker service must be active (running)

Registry Login (Host & Server)​

Both machines must be authenticated in the container registry before pulling or pushing images.

docker login

Settings Configuration​

The main deployment configuration is defined in settings.yaml.

It controls enabled blockchain networks and Docker images used for running the gateway services.

Chains Configuration​

The chains section defines which blockchain networks are enabled in the system and how they are accessed.

  • If a chain is not specified, it is considered disabled by default
  • If enabled: false, the chain is explicitly disabled
  • If enabled: true, an rpcUrl must be provided for the node connection

Example​

chains:
  tron:
    enabled: false
    rpcUrl: "https://api.trongrid.io"

When enabled, the system will use the provided RPC endpoint to interact with the blockchain network.

Deployment Configuration​

Example​

deployment:
  backend:
    image: "shuriken0x/zero-pay-backend"
  frontend:
    image: "shuriken0x/zero-pay-frontend"

Environment Variables​

The system requires a small set of environment variables for authentication, session management, and security.

These values are defined in .env and are critical for secure operation of the gateway.

Required Secrets​

The following variables must be configured:

  • SESSION_SECRET — used for securing user sessions and internal session encryption
  • JWT_SECRET_KEY — private key used for signing JWT tokens
  • JWT_PUBLIC_KEY — public key used for verifying JWT tokens

These values must be treated as sensitive and should never be exposed publicly.

Generating Secrets​

To simplify setup, the system provides an automated command for generating secure values:

nx run scripts:generate-secrets

This command generates a .env.secrets file containing all required cryptographic values.

Setup Process​

After generating secrets:

  1. Open the generated .env.secrets file
  2. Copy values into your .env.production file:
  3. Ensure .env.secrets is not committed to version control

Deployment Steps​

1. Build Images​

nx run scripts:build-images

2. Deploy to Server​

nx run scripts:deploy \
	--host="<ip>" --user="root" --password="<password>" \
	--domains="<domain>" --subdomains="metrics" \
	--deployment-type="remote" --no-logging \
	.env.production

Note​

Make sure DNS A records are configured before deployment:

  • <your-domain> → <server-ip>
  • metrics.<your-domain> → <server-ip>
