Forwarding

Forwarding is an automated mechanism that periodically consolidates funds from active wallets and transfers eligible balances to configured cold wallet addresses.

It is designed to reduce operational risk by minimizing funds held in hot wallets and ensuring controlled, rule-based asset movement across supported token and network pairs.

Key Features​

  • Scheduled asset sweeping (configurable interval)
  • User-level configuration
  • Automatic transfers to cold wallets
  • Reduces hot wallet exposure and operational risk

Forwarding Trigger Logic​

Forwarding is executed only when both conditions are satisfied simultaneously. These rules act as a single combined filter and define whether the forwarding process is started at all.

The evaluation scope is always per token + network pair (e.g. USDT / Ethereum). All balances are aggregated within this scope.

Trigger Conditions​

Forwarding will start only if:

  • Minimum wallet balance is satisfied for eligible wallets
  • Minimum total balance (token + network) is satisfied for the entire aggregated balance

If the total balance condition is not met, forwarding is not executed at all — even if some wallets individually exceed the minimum threshold.

If it is met, forwarding is executed only for wallets that pass the per-wallet balance filter.

Execution Model​

  1. The system calculates total balance per token + network
  2. It checks if the total balance exceeds the configured threshold
  3. If not — forwarding is skipped entirely
  4. If yes — wallets are filtered by minimum wallet balance
  5. Only eligible wallets are included in forwarding

Example​

  • 10 wallets on Ethereum network holding USDT
  • Balances:
    • 2 wallets: $2
    • 8 wallets: $10+

Configuration:

  • Minimum wallet balance: $5
  • Minimum total balance (USDT / Ethereum): $50

Result:

  • Total balance condition is satisfied → forwarding is triggered
  • Only 8 wallets (≥ $5) are included
  • 2 wallets (< $5) are excluded

When Forwarding Does NOT Run​

If the aggregated balance for a token + network pair is below the configured threshold:

  • Forwarding is not triggered
  • No wallets are processed
  • No transactions are created
